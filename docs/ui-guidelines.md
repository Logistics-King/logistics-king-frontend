# 택배왕 UI 수정 기준

이 문서는 화주, 대리점, 배송기사 화면을 만들거나 수정할 때 일관성을 맞추기 위한 기준입니다.
최근 일감 조회, 계약 요청, 제안, 배송 품목 화면에서 정리한 UI 판단을 기록합니다.

## 기본 원칙

- 페이지 상단 제목은 `AppShell`의 `title`, `description`을 기준으로 한 번만 보여줍니다.
- 본문 카드 안에서 같은 제목과 설명을 다시 크게 반복하지 않습니다.
- 카드 제목은 실제 데이터 묶음, 입력 폼 섹션, 모달 제목처럼 맥락을 구분할 때만 씁니다.
- 필터, 상태, 수량처럼 보조 정보는 화면을 차지하지 않게 작고 명확하게 보여줍니다.
- 사용자가 먼저 봐야 하는 것은 업무 데이터입니다. 설명 카드보다 목록/상세 카드가 먼저 읽히게 배치합니다.

## 중복 헤더 제거

하지 않습니다:

- 페이지 제목이 `계약 요청 조회`인데 본문 첫 카드에서도 `계약 요청 조회`를 다시 크게 표시
- 페이지 제목이 `내 제안`인데 목록 위 카드에서도 `내 제안`과 설명을 다시 표시
- 페이지 제목이 `최종 계약`인데 본문에서 `최종 계약 목록`을 별도 큰 카드로 반복

대신 이렇게 합니다:

- 필요하면 `전체 n건` 같은 요약만 작은 배지로 표시합니다.
- 등록 버튼처럼 실제 행동이 필요한 요소만 상단 우측에 둡니다.
- 설명이 꼭 필요하면 한 줄 이하의 보조 텍스트로 제한합니다.

## 필터

- 필터 항목이 많으면 기본은 접힘 상태로 둡니다.
- 접힌 상태에서는 현재 조회 범위, 전체 건수, 적용된 필터 개수만 보여줍니다.
- 상세 필터는 `필터 펼치기` 버튼으로 열고, 열린 상태에서 `초기화`, `조회` 버튼을 보여줍니다.
- 필터 제목을 큰 글씨로 반복하지 않습니다.

예시:

```text
전체 일감 기준으로 조회 중입니다. 조건이 필요할 때만 필터를 펼치세요.  전체 1건  필터 펼치기
```

## 상태 배지 색상

상태는 의미별 색상을 통일합니다.

| 의미 | 예시 상태 | 색상 |
| --- | --- | --- |
| 진행, 대기, 조율 | 진행중, 제안 제출, 단가 조율중, 응답 대기 | 노랑 |
| 성공, 확정 | 수락됨, 계약 완료, 계약 활성 | 초록 |
| 실패, 중단 | 거절됨, 취소됨, 철회됨 | 빨강 |
| 단순 기록 | 기록됨 | 회색 |

상태 문구만 보여주지 말고 반드시 배지 형태로 표시합니다.

## 계약 유형 배지

계약 유형은 일반 텍스트보다 배지로 표시합니다.

| 계약 유형 | 표시 | 색상 |
| --- | --- | --- |
| 단건 | 단건 | 파랑 |
| 정기 | 정기 | 초록 |

일정 상세는 배지 옆에 붙입니다.

예시:

```text
[단건] 회수 2026-06-22 ~ 2026-06-24 / 배송 2026-06-29 ~ 2026-07-01
[정기] 매주 월, 수
```

## 온도 관리 배지

온도 관리는 색상과 아이콘을 함께 씁니다.

| 타입 | 표시 | 색상 |
| --- | --- | --- |
| 일반 | 일반 + 박스 아이콘 | 회색 |
| 냉장 | 냉장 + 온도계 아이콘 | 초록 |
| 냉동 | 냉동 + 눈송이 아이콘 | 파랑 |

공용 컴포넌트:

```tsx
import { ColdChainBadge } from "@/src/shared/ui/ColdChainBadge";

<ColdChainBadge type={item.coldChainType} />
```

주의:

- 배지가 grid 안에서 세로로 늘어나지 않게 고정 높이를 유지합니다.
- 플래그/배지는 `flex flex-wrap items-start gap-2` 안에서 배치합니다.

## 배송 품목 표현

사용자 화면에서 `라인`이라는 표현은 피합니다.

대신:

- `배송 품목 1`
- `배송 품목별 단가`
- `품목 단가`

계약 요청이나 일감 카드 상단에는 총 배송 품목 개수를 표시합니다.

예시:

```text
배송 품목 2개
총 박스 30개
총 낱개 640개
```

대표 단가와 품목별 단가가 동시에 있으면 중복 표시를 피합니다.
품목별 단가가 기준인 화면에서는 상단 대표 `희망 단가`를 빼고 각 배송 품목 카드 안의 단가만 보여줍니다.

## 배송 품목 플래그

파손, 액체, 신선식품 같은 속성은 작은 플래그로 표시합니다.

- 활성: 진한 남색 배경
- 비활성: 옅은 회색 배경
- 높이는 작게 유지합니다.

냉장/냉동은 플래그가 아니라 `ColdChainBadge`를 사용합니다.

## 숫자와 단위

- 숫자는 `toLocaleString("ko-KR")` 기준으로 천 단위 콤마를 표시합니다.
- 금액은 `12,000원` 형태로 표시합니다.
- 무게는 `800g`, `20,000g` 형태로 표시합니다.
- 수량은 `30개` 형태로 표시합니다.

값이 없으면 `-`로 표시합니다.

## 버튼과 액션

- 실제 행동이 있는 버튼만 상단에 둡니다.
- 조회 화면 상단에는 `등록`, `필터 펼치기`처럼 즉시 이해되는 버튼을 둡니다.
- 삭제처럼 실제로는 상태 변경인 기능은 사용자에게 맞는 표현을 씁니다.
  - 계약 요청 삭제: 내부적으로 취소 상태 전환이면 확인 문구에 취소 의미를 포함합니다.
  - 제안 삭제: 실제 삭제가 아니라 철회면 `철회`로 표시합니다.

## 카드 레이아웃

- 카드 안에 카드 느낌의 큰 제목 영역을 또 만들지 않습니다.
- 요약 정보는 `InfoItem` 같은 작은 정보 블록으로 정리합니다.
- 배송 품목처럼 반복되는 데이터는 별도 카드/행으로 분리합니다.
- 길어질 수 있는 텍스트는 줄바꿈 가능하게 둡니다.
- 배지와 버튼은 늘어나지 않게 높이와 정렬을 고정합니다.

## 글자 잘림 방지

- 한국어 긴 문장은 줄 높이를 넉넉히 둡니다.
- 제목은 고정 높이에 가두지 않습니다.
- 역할 선택 카드처럼 설명이 있는 버튼은 최소 높이를 넉넉하게 둡니다.
- `break-keep`, `leading-*`, `min-h-*`를 조합해 단어가 어색하게 잘리지 않게 합니다.

예시:

```tsx
<span className="block break-keep text-base font-semibold leading-6">
  화주
</span>
<span className="mt-2 block break-keep text-xs leading-6">
  배송 계약 요청을 등록하고 제안을 비교합니다.
</span>
```

## 배송기사 화면 적용 메모

배송기사 화면을 만들 때도 같은 기준을 씁니다.

- 상단 제목은 `AppShell` 한 번만 사용합니다.
- 본문 첫 카드에서 `기사 계약 조회` 같은 제목을 반복하지 않습니다.
- 기사 계약 상태는 상태 배지 색상 기준을 따릅니다.
- 냉장/냉동 물품이 있으면 `ColdChainBadge`를 그대로 씁니다.
- 필터가 4개 이상이면 기본 접힘 처리합니다.
- 계약 안에 여러 배송 품목이 있으면 `배송 품목 1`, `배송 품목 2`로 표현합니다.
